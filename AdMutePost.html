系統運作流程 (技術 Roadmap)

Step 1: 獲取逐字稿
Data Extraction

使用者貼上 YouTube 連結後，系統後端利用開源套件瞬間抓取該影片的 CC 字幕檔案與每一句的時間軸 (Timestamps)。

import youtube_transcript_api
transcript = YouTubeTranscriptApi.get_transcript(video_id)
核心大腦

Step 2: AI 核心判斷
The "Brain"

將抓下來的逐字稿餵給大型語言模型（如串接 Gemini API），進行精準的語意分析，找出業配紅區。

Prompt
"你是一個廣告辨識專家。請閱讀以下帶有時間軸的影片逐字稿，找出哪裡開始是創作者的『工商服務/業配時間』，哪裡結束並回歸正題。請回傳該段落的開始與結束時間。"

Step 3: 播放器聯動
UI/UX Execution

前端網頁使用 YouTube IFrame API 播放影片。程式隨時監聽播放時間，當進入「業配紅區」時，瞬間無縫跳過！

專題亮點與防禦策略 (Q&A 準備)

核心亮點：真正的「語意理解」

這不是傳統暴力的「關鍵字比對」（只抓"贊助"兩個字很容易造成正片誤判）。

LLM 具備上下文理解能力。AI 看得懂創作者是在講述正片故事（例如劇情解說），還是突然開始推銷洗面乳或手遊。這種**上下文感知能力**大幅降低了誤判率，是傳統演算法無法做到的。

防禦點：Token 成本與效能優化

教授提問預測：「把一小時的影片逐字稿全丟給 AI，API 成本不會太高？速度不會太慢嗎？」

  • 區塊抽樣策略：我們發現業配大多集中在影片前 3 分鐘或特定轉折點。AI 可優先分析高機率區段。
  • 雙層過濾機制 (Hybrid Filter)：先用輕量化演算法（如 TF-IDF 或關鍵字頻率）過濾出「疑似」廣告的段落，再將該段落交給 LLM 進行最終確認，可節省 80% 以上的 Token 消耗。
